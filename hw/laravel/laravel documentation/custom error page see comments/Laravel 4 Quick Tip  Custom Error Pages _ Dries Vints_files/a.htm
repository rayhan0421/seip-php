                        {
                        // log the error
                        Log::error($exception);

                        // check environment
                        if(! App::environment() == 'development'){
                        switch ($code)
                        {
                        case 404:
                        $message = Lang::get('errors.404');
                        break;
                        case 500:
                        $message = Lang::get('errors.500');
                        break;
                        default:
                        $message = Lang::get('errors.default');
                        }

                        return View::make('pages.errors.default')
                        ->with('error', $code)
                        ->with('message', $message);
                        }

                        });

                                                                                                              Jason Lewis 2 years ago

                                                                                                              You can actually type hint the exception that's thrown instead of using a switch.

                                                                                                              App::error(function(Symfony\Component\HttpKernel\Exception\NotFoundHttpException $exception, $code)
{
    return View::make('errors.404');
});

                                                                                                              Obviously that's a bit of an eye sore, so either inject the exception into the global namespace with use or, luckily for us, Laravel provides an App::missing() method.

                                                                                                              App::missing(function($error)
{
    return View::make('errors.404');
});

                                                                                                              This isn't limited to just that exception. You can catch any number of exceptions with App::error() and tailor each one differently. But the exceptions stack, so be sure to define the handler for Exception last.
