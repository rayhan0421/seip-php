                                                      fideloper Mod Morgon • 2 years ago

                                                      Hi -

                                                      You're definitely not limited to one binding per Service Provider - You can add as many as you need.

                                                      I typically separate out my SP's per "module" (I use that term loosely - basically per "high level namespace"). For instance I'll use one SP for Anvil and one for TNT (as you can see above).

                                                      Throw in as many bindings as you need in the register() method :D

                                                      Note: This doesn't sound like your situation, but If you find yourself loading in classes at run-time based on some input or other factors, you might consider using a Factory of some sort - a class which can take inputs and can decide what class/object to return to your code (That's not Laravel specific, just coding in general).

                                                      Hope that helps
